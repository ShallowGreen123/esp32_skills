# BQ25896 + BQ27220 稳定工作流

## 1. 本板约束

基于当前仓库里的板级资料：

- I2C: `SDA=GPIO8`, `SCL=GPIO18`
- `BQ27220` 地址: `0x55`
- `BQ25896` 地址: `0x6B`
- 电池记录: `3.7V 1300mAh`
- `PN532 (0x24)` 与两颗电源芯片共用同一条 I2C

参考：

- [hardware-analysis.md](../hardware/notes/hardware-analysis.md)
- [pinmap.md](../hardware/pinmap/pinmap.md)

## 2. 两颗芯片的分工

- `BQ25896`: 充电器 + power-path。负责输入限流、预充、恒流、恒压、终止、再充、系统最小电压。
- `BQ27220`: 电量计。负责 `SOC/FCC/SOH`、温度/电流/电压跟踪、满充判定、休眠/relax 状态。

要稳定，关键不是“两个芯片都能读到”，而是它们对同一块电池使用同一组参数：

- `BQ25896.VREG` 要和 `BQ27220.Charging Voltage` 一致
- `BQ25896.ICHG` 要和 `BQ27220.Charging Current` 同量级
- `BQ25896.ITERM` 要和 `BQ27220.Taper Current` 同量级

如果这三组不对齐，最常见的现象是：

- 满充识别不稳定
- 拔掉 USB 后 `SOC` 突跳
- 长时间学不准 `FCC`
- 明明“充满”，但 gauge 不置 `FC`

## 3. 上电初始化总顺序

推荐顺序：

1. 主控上电后先初始化 I2C，不要立刻扫全总线，先只探测 `0x55/0x6B/0x24`
2. 先读一次 `BQ25896` 状态寄存器和 fault 寄存器，确认不是上次异常残留
3. 临时关闭 `BQ25896` 充电使能，避免边改寄存器边充电
4. 先把 `BQ25896` 的 watchdog、输入限流、充电电流、电压、终止条件配置好
5. 再配置 `BQ27220` 的 data memory
6. 配完 `BQ27220` 后确认 `OperationStatus()[CFGUPDATE] = 0`
7. 重新使能 `BQ25896` 充电
8. 运行期周期性读取：
   - `BQ25896`: 充电状态、fault、ADC
   - `BQ27220`: `Voltage/Current/RemainingCapacity/FCC/SOC/BatteryStatus/OperationStatus`

推荐做成两个阶段：

- `bring-up`: `BQ25896` 关闭 watchdog，关闭 safety timer，打开 1s ADC，方便看日志
- `production`: `BQ25896` watchdog 仍建议关闭，safety timer 视实际充电时长决定是否开启

## 4. BQ25896 推荐初始参数

下面这组值适合先把 `1300mAh` 的 `4.2V` 单节 LiPo 稳定跑起来。它不是最终量产值，但足够作为第一版“稳态参数”。

### 4.1 参数表

| 项目 | 寄存器/字段 | 建议值 | 说明 |
|---|---|---:|---|
| 输入限流 | `REG00.IINLIM` | `500mA` 起步 | 先保守，确认输入源能力后再放大 |
| ILIM 引脚 | `REG00.EN_ILIM` | `1` | 如果板上 ILIM 电阻可信，保留硬件上限；若怀疑被卡流，可临时清零排查 |
| 最小系统电压 | `REG03.SYS_MIN` | `3.5V` | 默认值，适合先跑通系统 |
| 快充电流 | `REG04.ICHG` | `640mA` | 约 `0.5C`，对 `1300mAh` 电池较稳 |
| 预充电流 | `REG05.IPRECHG` | `128mA` | 不要留太大，减少低电压电芯压力 |
| 终止电流 | `REG05.ITERM` | `128mA` | 必须与 `BQ27220` 的 `Taper Current` 同量级 |
| 充电电压 | `REG06.VREG` | `4.208V` | TI 这档最接近 `4.2V` |
| 低电压到快充阈值 | `REG06.BATLOWV` | `3.0V` | 默认即可 |
| 再充阈值 | `REG06.VRECHG` | `100mV` | 先用默认值 |
| 充电终止开关 | `REG07.EN_TERM` | `1` | 建议开启 |
| I2C watchdog | `REG07.WATCHDOG` | `Disable` | 最稳，避免 40s 后寄存器回默认 |
| 安全 timer | `REG07.EN_TIMER` | `0` bring-up / `1` production | 调试先关，量产再决定 |
| 快充 timer | `REG07.CHG_TIMER` | `12h` | 若启用 safety timer，先用默认 12h |
| ADC | `REG02.CONV_RATE` | `1s continuous` | 调试阶段建议开 |
| 自动输入检测 | `REG02.AUTO_DPDM_EN` | `1` | 若输入口按 TI 参考接法，可保留 |
| ICO | `REG02.ICO_EN` | `0` 起步 | 先稳定再启用，避免弱输入源下状态复杂 |
| TMR2X | `REG09.TMR2X_EN` | `1` | 若启用 safety timer，DPM/热调节时自动放慢计时 |
| BATFET | `REG09.BATFET_DIS` | `0` | 正常工作必须保持允许导通 |

### 4.2 一版可直接写入的寄存器值

下面是基于上面参数推出来的一组首版值：

| 寄存器 | 值 | 含义 |
|---|---|---|
| `REG00` | `0x48` | `EN_HIZ=0`, `EN_ILIM=1`, `IINLIM=500mA` |
| `REG02` | `0x41` | `CONV_RATE=1`, `ICO_EN=0`, `AUTO_DPDM_EN=1` |
| `REG03` | `0x4A` | `CHG_CONFIG=0`, `SYS_MIN=3.5V`, 并顺手 `WD_RST=1` |
| `REG04` | `0x0A` | `ICHG=640mA` |
| `REG05` | `0x11` | `IPRECHG=128mA`, `ITERM=128mA` |
| `REG06` | `0x5E` | `VREG=4.208V`, `BATLOWV=3.0V`, `VRECHG=100mV` |
| `REG07` | `0x85` | `EN_TERM=1`, `WATCHDOG=Disable`, `EN_TIMER=0`, `CHG_TIMER=12h` |
| `REG09` | `0x44` | `TMR2X_EN=1`, `BATFET_DIS=0`, `BATFET_RST_EN=1` |
| `REG03` | `0x5A` | 最后一步重新 `CHG_CONFIG=1` 使能充电 |

说明：

- 如果你确认量产要开 safety timer，可把 `REG07` 从 `0x85` 改为 `0x8D`
- 如果你怀疑板上的 `ILIM` 电阻把输入流限住，可把 `REG00` 从 `0x48` 改为 `0x08`
- 如果输入不是标准 USB 口，而是固定 5V 适配器，可考虑把 `AUTO_DPDM_EN` 关掉并手动设 `IINLIM`

## 5. BQ27220 推荐初始参数

`BQ27220` 的准确度主要取决于两件事：

1. 电池参数有没有和真实电芯匹配
2. charger 的终止条件有没有和 gauge 的满充判定对齐

如果手上没有电芯厂给的 CEDV 参数，建议先用 TI 预置 chemistry 跑通，再通过 `bqStudio + GAUGEPARCAL` 生成最终配置。

### 5.1 必配参数

| 项目 | Data Memory 地址 | 建议值 | 说明 |
|---|---|---:|---|
| Full Charge Capacity | `0x929D` | `1300mAh` | 初始值先与设计容量一致，后续允许学习更新 |
| Design Capacity | `0x929F` | `1300mAh` | 必改，默认是 `3000mAh` |
| Design Voltage | `0x92A3` | `3700mV` | 单节 LiPo 典型值 |
| Charge Termination Voltage | `0x92A5` | `100mV` | 这是 gauge 侧的 taper-voltage 等效项 |
| Charging Current | `0x91FB` | `640mA` | 要与 charger 一致 |
| Charging Voltage | `0x91FD` | `4200mV` | 要与 charger 一致 |
| Taper Current | `0x9201` | `128mA` | 要与 `BQ25896.ITERM` 同量级 |

### 5.2 低功耗与状态切换参数

默认值不一定适合 ESP32 类产品。建议按“实测电流”调，不要死抄默认。

| 项目 | 地址 | 默认 | 初始建议 |
|---|---|---:|---:|
| Sleep Voltage Time | `0x921E` | `20s` | `20s` 起步 |
| Sleep Current Time | `0x921F` | `20s` | `20s` 起步 |
| Discharge Detection Threshold | `0x9228` | `60mA` | `20mA` 到 `30mA` |
| Charge Detection Threshold | `0x922A` | `75mA` | `30mA` 到 `50mA` |
| Quit Current | `0x922C` | `40mA` | `10mA` 到 `20mA` |
| Discharge Relax Time | `0x922E` | `60s` | `60s` |
| Charge Relax Time | `0x9230` | `60s` | `60s` |
| Quit Relax Time | `0x9231` | `1s` | `3s` |

调参原则：

- `Quit Current` 要高于系统真实待机电流，但要低于 `Dsg/Chg Threshold`
- 如果你在电池供电下常见电流只有 `10mA` 到 `30mA`，默认 `60mA/75mA` 太高，gauge 很难正确进入 `DISCHARGE/CHARGE`
- 如果 `SOC` 在轻载场景下变化很怪，先检查这三项：`Dsg Threshold / Chg Threshold / Quit Current`

一个实用起点：

- 深睡电流 `< 5mA`: `Quit=10mA`, `Dsg=20mA`, `Chg=30mA`
- 轻待机电流 `10mA` 到 `15mA`: `Quit=15mA`, `Dsg=25mA`, `Chg=35mA`
- 如果系统经常有短脉冲负载，把 `Quit Relax Time` 从 `1s` 提到 `3s`

## 6. BQ27220 的写入流程

### 6.1 必须知道的点

- `BQ27220` 不是传统 subclass/block 写法，而是 direct address 写法
- 地址写到 `0x3E/0x3F`
- 参数数据跟在后面自动递增
- 校验和写到 `0x60`
- 总长度写到 `0x61`
- 地址是 little-endian，参数值是 big-endian

### 6.2 推荐配置顺序

1. `UNSEAL`
2. `FULL ACCESS`
3. `ENTER_CFG_UPDATE`
4. 依次写 `0x929D/0x929F/0x92A3/0x92A5/0x91FB/0x91FD/0x9201`
5. 写当前项目需要的阈值参数
6. `EXIT_CFG_UPDATE`
7. 读取 `OperationStatus(0x3A/0x3B)`，确认 `CFGUPDATE=0`
8. `SEAL`

### 6.3 常用 control 子命令

| 功能 | 写法 |
|---|---|
| Unseal 第 1 步 | `w 0x55 0x00 0x14 0x04` |
| Unseal 第 2 步 | `w 0x55 0x00 0x72 0x36` |
| Full Access 第 1 步 | `w 0x55 0x00 0xFF 0xFF` |
| Full Access 第 2 步 | `w 0x55 0x00 0xFF 0xFF` |
| Enter Config Update | `w 0x55 0x00 0x90 0x00` |
| Exit Config Update | `w 0x55 0x00 0x92 0x00` |
| Seal | `w 0x55 0x00 0x30 0x00` |

### 6.4 直接写 `Design Capacity = 1300mAh` 的例子

`1300 = 0x0514`

- 地址: `0x929F`，按 little-endian 写成 `9F 92`
- 参数: 按 big-endian 写成 `05 14`
- 校验和: `0xFF - ((0x9F + 0x92 + 0x05 + 0x14) & 0xFF) = 0xB5`
- 总长度: `2 字节地址 + 2 字节数据 + 1 字节校验和 + 1 字节长度 = 0x06`

示例：

```text
w 0x55 0x00 0x90 0x00
w 0x55 0x3E 0x9F 0x92 0x05 0x14
w 0x55 0x60 0xB5 0x06
w 0x55 0x00 0x92 0x00
```

### 6.5 建议做一个通用 helper

建议固件里统一用一个 helper：

```cpp
bool bq27220WriteDmU16(uint16_t addr, uint16_t value_be);
```

内部流程固定成：

1. `write(0x3E, addr_lsb, addr_msb, value_msb, value_lsb)`
2. `checksum = 0xFF - ((addr_lsb + addr_msb + value_msb + value_lsb) & 0xFF)`
3. `write(0x60, checksum, 0x06)`
4. 适当延时 `5ms` 到 `20ms`
5. 读回确认

## 7. Arduino 侧建议的伪代码

```cpp
void powerInit() {
    Wire.begin(8, 18);
    Wire.setClock(400000);

    // 1) Charger first: stop charging while reconfiguring.
    bq25896WriteReg(0x03, 0x4A);
    bq25896WriteReg(0x07, 0x85);
    bq25896WriteReg(0x00, 0x48);
    bq25896WriteReg(0x04, 0x0A);
    bq25896WriteReg(0x05, 0x11);
    bq25896WriteReg(0x06, 0x5E);
    bq25896WriteReg(0x02, 0x41);
    bq25896WriteReg(0x09, 0x44);

    // 2) Gauge next.
    bq27220Unseal();
    bq27220FullAccess();
    bq27220EnterCfgUpdate();

    bq27220WriteDmU16(0x929D, 1300); // FCC init
    bq27220WriteDmU16(0x929F, 1300); // Design Capacity
    bq27220WriteDmU16(0x92A3, 3700); // Design Voltage
    bq27220WriteDmU16(0x92A5, 100);  // Charge Termination Voltage
    bq27220WriteDmU16(0x91FB, 640);  // Charging Current
    bq27220WriteDmU16(0x91FD, 4200); // Charging Voltage
    bq27220WriteDmU16(0x9201, 128);  // Taper Current

    // Thresholds: tune after real current measurement.
    bq27220WriteDmU16(0x9228, 20);   // Dsg threshold
    bq27220WriteDmU16(0x922A, 30);   // Chg threshold
    bq27220WriteDmU16(0x922C, 10);   // Quit current

    bq27220ExitCfgUpdate();
    bq27220Seal();

    // 3) Enable charging after both chips agree on the same battery settings.
    bq25896WriteReg(0x03, 0x5A);
}
```

## 8. 运行期监控建议

每 `1s` 到 `5s` 读取一次：

- `BQ25896`
  - 充电状态
  - fault
  - 电池电压 / 系统电压 / VBUS / 充电电流
- `BQ27220`
  - `Voltage()` `0x08/0x09`
  - `Current()` `0x0C/0x0D`
  - `RemainingCapacity()` `0x10/0x11`
  - `FullChargeCapacity()` `0x12/0x13`
  - `StateOfCharge()` `0x2C/0x2D`
  - `BatteryStatus()` `0x0A/0x0B`
  - `OperationStatus()` `0x3A/0x3B`

重点观察：

- `BQ25896` 是否经常落入 `DPM_STAT`
- `BQ25896` 是否偶发 fault 或寄存器回默认
- `BQ27220` 是否进入 `CFGUPDATE`
- `BQ27220` 的 `FC/CHGINH/SLEEP/EDV2/SEC[1:0]` 是否符合预期

## 9. 最容易踩的坑

### 9.1 `BQ25896` watchdog 没关

默认 `40s`。如果主控不定期喂狗，很多寄存器会回默认，看起来像“配置会自己丢”。

### 9.2 只改 charger，不改 gauge

最典型的错误是：

- `BQ25896.ITERM = 128mA`
- `BQ27220.Taper Current = 100mA`

这样满充判定经常对不上。

### 9.3 `BQ27220` 只改了 `Design Capacity`

只改容量，不改 `Charging Current / Charging Voltage / Taper Current / Charge Termination Voltage`，SOC 大概率还是会跳。

### 9.4 轻载场景仍使用默认 `60mA/75mA`

如果产品在电池模式下经常只有十几毫安，默认阈值太高，会导致 gauge 不进入你以为的状态。

### 9.5 没有生产态 reload 机制

如果你的最终方案不是 OTP 固化，而是 RAM 配置，那么每次 `POR` 后都要由 MCU 重新下发完整配置。

## 10. 第一轮验收标准

满足下面 5 条，基本可以认为“第一版稳定”：

1. 插拔 USB 后 `BQ25896` 寄存器不莫名回默认
2. 充电时 `VBUS/SYS/BAT` 变化平滑，不频繁进出 DPM
3. 满充时 `BQ27220` 能稳定拉到 `FC=1`
4. 拔掉 USB 后 `SOC` 不会瞬间大跳
5. 完整做一轮 `充满 -> 静置 -> 放电 -> 静置 -> 再充满` 后，`SOC/FCC` 曲线单调且可复现
